JDesktopPane

According to the Java API, JDesktopPane is "used to create a multiple-document interface or a virtual desktop". It's like if you had multiple windows open on your computer, except they're all contained in one window. This makes it more organized if the windows are all related, like if you had multiple drawing windows but only one universal brush tool and colour mixer.

Let's try creating an empty JDesktopPane and containing it in a JFrame: 

import java.awt.BorderLayout;
import javax.swing.JDesktopPane;
import javax.swing.JFrame;

public class DesktopDemo extends JFrame 
{
    private JDesktopPane desktop;
    
    public DesktopDemo()
    {
        super( "Verndows 10" );
        setLayout( new BorderLayout(10, 10) ); // Set frame layout
        
        // Initialize panels and widgets
        desktop = new JDesktopPane();
        
        // Add panels to main frame
        add( desktop, BorderLayout.CENTER );
    }
    
} // end class DesktopDemo

And the test class: 


import javax.swing.JFrame;

public class DesktopTest
{
    public static void main( String args[] )
    {
        DesktopDemo desktop = new DesktopDemo();  
        desktop.setDefaultCloseOperation( JFrame.EXIT_ON_CLOSE );
        desktop.setSize( 500, 420 );
        desktop.setVisible( true );
    }
}

JInternalFrame

The JInternalFrame is functionally identical to a JFrame - you can set the layout, add widgets, handle events, etc. - with the difference that it can only be moved around within the space of the JDesktopPane.

Let's see what a DrawFrame object would look like as a JInternalFrame, and throw in a JMenuBar and JToolbar for fun:

Test Code 

import java.awt.BorderLayout;
import java.awt.FlowLayout;
import javax.swing.JButton;
import javax.swing.JDesktopPane;
import javax.swing.JInternalFrame;
import javax.swing.JFrame;
import javax.swing.JLabel;
import javax.swing.JMenu;
import javax.swing.JMenuBar;
import javax.swing.JPanel;
import javax.swing.JToolBar;

public class DesktopDemo extends JFrame 
{
    private JPanel topJPanel;
    private JDesktopPane desktop;
    private JPanel bottomJPanel;
    private JInternalFrame drawTest;
    private JMenuBar mainBar;
    private JMenu fileMenu, editMenu, viewMenu; // Menus
    private JToolBar toolbar;
    private JButton report;
    
    public DesktopDemo()
    {
        super( "Verndows 10" );
        setLayout( new BorderLayout( 10, 10 ) ); // Set frame layout
        
        // Initialize panels and widgets
        desktop = new JDesktopPane();
        
        mainBar = new JMenuBar();
        setJMenuBar( mainBar );
        
        fileMenu = new JMenu( "File" );
        editMenu = new JMenu( "Edit" );
        viewMenu = new JMenu( "View" );
        
        drawTest = new JInternalFrame( "Paint", true, true, true, true );
        drawTest.setSize( 400, 400 );
        drawTest.setVisible( true );
        
        toolbar = new JToolBar( "Options" );
        report = new JButton( "Report Bugs" );
        
        // Add widgets to panels
        drawTest.add( new DrawPanel( new JLabel( "(0, 0)") ) );
        desktop.add( drawTest );
        add( toolbar, BorderLayout.WEST );
  
        mainBar.add( fileMenu );
        mainBar.add( editMenu );
        mainBar.add( viewMenu );
        
        toolbar.add( report );
        
        // Add panels to main frame
        add( desktop, BorderLayout.CENTER );
        
    }   

} // end class DesktopDemo

The test class remains the same.

Considerations

  • JInternalFrame's constructor takes up to four boolean arguments; resizable, closable, maximizable, and iconifiable (minimizable) in order. They default to false.
  • You must call the setSize() and setVisible() methods of each JInternalFrame object in order to see it.
  • You cannot nest JFrames (or JInternalFrames), so you'll have to use JPanels instead.
  • Remember to consult the Java API for more information on each class.
